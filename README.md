# Q&A API Service

API-сервис для вопросов и ответов на Go с использованием PostgreSQL.

## Функциональность

- Создание, получение и удаление вопросов
- Добавление ответов к вопросам
- Получение и удаление ответов
- Каскадное удаление ответов при удалении вопроса

## Технологии

- Go 1.21+
- PostgreSQL
- GORM
- Goose (миграции)
- Docker & Docker Compose

## Запуск проекта

### Требования
- Docker
- Docker Compose

### Запуск

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd qna-api
2. Запустите приложение:

```bash
docker-compose up --build
Приложение будет доступно по адресу: http://localhost:8080
Примеры запросов
Создание вопроса
bash
curl -X POST http://localhost:8080/questions \
  -H "Content-Type: application/json" \
  -d '{"text": "Как работает Go?"}'
Добавление ответа
bash
curl -X POST http://localhost:8080/questions/1/answers \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user-123",
    "text": "Go - это компилируемый язык программирования"
  }'
Миграции
Миграции выполняются автоматически при запуске через Docker Compose.

Тестирование
Для запуска тестов:

bash
go test ./...
Структура проекта
Проект следует чистой архитектуре с разделением на:

handlers - HTTP обработчики

services - бизнес-логика

repositories - работа с базой данных

models - структуры данных

config - конфигурация приложения

text

## Ключевые особенности реализации:

1. **Чистая архитектура** - разделение на слои (handler, service, repository)
2. **Каскадное удаление** - ответы автоматически удаляются при удалении вопроса
3. **Валидация** - проверка входных данных
4. **Миграции** - использование Goose для управления схемой БД
5. **Docker** - полная контейнеризация приложения
6. **Тесты** - пример unit-теста с testify
7. **Логика** - соблюдение всех бизнес-правил

Проект готов к использованию и может быть легко расширен дополнительной функциональностью.
